# Atomisation queue

Ordered queue of topic groups still to atomise, built from `TRIAGE.md` after
excluding the 20 already-audited Stage 4 topics. A **group** is every
surviving booklet for one topic (OLD booklets marked KEEP, plus any
NEW-origin booklet that needs atomising even though it superseded an OLD
file) — reviewed together as **one combined proposal**, per the booklet
atomisation workflow.

**One group per session.** Work top to bottom. After a group's proposal is
approved and applied (`npm run validate` clean), flip its status
`pending → applied` (or `→ nil` if the pass recommended no changes) and move
on to the next `pending` row — do not skip ahead.

Target topic ids and dot-point ids are looked up from `data/topics.json` /
`data/dotpoints.json` by topic-name match. Where a target is genuinely
ambiguous (no matching topic exists yet), it's marked `TBC` — resolve that at
review time, don't guess.

**Ordering rationale:**
1. **Stage 4 remainder** — the booklets not yet in the audited 20, in the
   order set by `README.md`: Ratios & Rates → Length → Volume → Properties
   of Geometrical Figures → Data Analysis → Data Visualisation → Probability.
2. **Stage 5** — number/financial → algebraic techniques → indices →
   equations → linear relationships → non-linear/functions/variation →
   trigonometry → measurement (area/surface area/volume) → geometry
   (similarity/congruence/circles) → data analysis → probability. Core (`A`)
   before Path (`B`/`C`) within the same topic family, since Path extends
   Core. Where OLD and NEW booklets align on a topic, both are listed in the
   group so the review pass reads OLD's still-uncovered content alongside
   NEW's rewrite.
3. **Stage 6** — Standard first (self-contained, no Stage-5 feeder), then
   Advanced, then Extension 1, each in intra-course dependency order
   (`data/topics.json`'s `order` field already reflects this for Stage 6 and
   is used directly). Year 11 before Year 12 within each course, per the
   Stage-6 course-split rules.
