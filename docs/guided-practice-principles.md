# Guided practice (We Do) principles

The **We Do** is the first question the learner attempts after the worked example,
with support. It bridges the worked example → independent practice. Sequence:
**worked example → We Do → practice (Foundation/Development/Mastery)**. Companion to
[worked-example-principles.md](worked-example-principles.md) and
[atomisation-teaching.md](atomisation-teaching.md).

> **Scope note:** pared to what informs **the site**. Classroom-only mechanics —
> mini-whiteboards, 80% live thresholds, peer marking, the tick trick — are dropped;
> kept is what shapes content + UI (side-by-side layout, step-by-step reveal).

## Choice of question (the golden rule)
Test the **same skill** as the worked example, but still require **some thinking**.
- **Too similar** (same numbers/pattern) → mindless copying, tests nothing.
- **Too different** (new twist, double-rounding, harder arithmetic) → tests untaught
  skills. The first We Do is **not** the place to add twists.
- Same underlying skill, **different surface features**, similar cognitive demand.

## Layout (site)
Keep the completed worked example **visible side-by-side** with the We Do so the
learner can reference the model while working. Reveal the We Do as a distinct step
after the worked example.

## Step-by-step reveal (site)
Don't present the whole solution at once. Structure the We Do as **one step at a
time**: prompt for a step → let the learner reveal/check that step → continue. This
mirrors the line-by-line worked example, gives per-step feedback, and prevents
cascading errors. (Drives the UI: each We Do stores ordered steps, each
independently revealable.)

## Sequence of We Dos (several short ones)
Prefer a short sequence of We Dos over a single one (a learner needs ~7 reps to reach
mastery), using three question kinds:
- **What If** — change **one** element from the previous item; focuses attention on
  that change.
- **Reset** — change several familiar elements at once; tests transfer with a fresh
  start.
- **Extension** — introduce **one** new element, low-stakes; probes readiness for the
  next skill.

Example (expand single brackets) — worked example `3(x + 4)`; We Do `5(x + 2)`
[What If], `4(y − 3)` [What If, subtraction], `2(3x + 1)` [Reset], `x(x + 5)`
[Extension].
